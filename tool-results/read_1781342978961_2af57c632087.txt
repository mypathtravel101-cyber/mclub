     1→# Cover Design V3.0 - Cover Layout Engine Specification
     2→
     3→> The cover is the first impression. Either skip it, or build it like architecture.
     4→
     5→---
     6→
     7→## ⚠️ Critical Rules (Read First)
     8→
     9→1. **Cover is OPTIONAL.** Do NOT force a cover on documents that don't need one. When in doubt, skip.
    10→2. **Unified cover system.** All routes (Report, Creative, Academic) use the HTML/Playwright cover system. Templates 01-07 are general-purpose; Templates 08-10 are academic-specific (dark backgrounds, scholarly typography); **Template 11 is institutional (white bg, black border frame, structured fields)**. All routes generate covers via Playwright and merge via pypdf.
    11→3. **Single PDF output.** Never deliver a separate cover PDF.
    12→   - **Creative route**: Cover is part of the same HTML document → single PDF output inherently.
    13→   - **Report route**: Cover PDF (Playwright) + body PDF (ReportLab) → merged via pypdf into one final PDF.
    14→   - **Academic route**: Cover PDF (Playwright) + body PDF (Tectonic) → merged via pypdf into one final PDF.
    15→4. **Page isolation.** Cover content must NEVER share a page with TOC, body text, or any subsequent content. Report/Academic: isolation is inherent in the merge pipeline. Creative: CSS page-break enforces isolation. Cover + TOC on the same page = **critical bug**.

    33→# PART 0: GLOBAL ENGINE ARCHITECTURE (Mandatory)
    34→
    35→These three architectural upgrades are **mandatory** for ALL cover rendering. They eliminate 90% of squished/misaligned/overflow bugs at the engine level.
    36→
    37→---
    38→
    39→## A0.0 - Global Base Parameters
    40→
    41→**All templates MUST obey these.**
    42→
    43→```
    44→W = page total width
    45→H = page total height
    46→U = W * 0.05           # Base spacing unit (5% of page width)
    47→                        # All spacing should be multiples of U
    48→```
    49→
    50→**Coordinate origin:** `(0, 0)` at top-left corner. Some PDF libraries (ReportLab) use bottom-left origin - invert Y axis accordingly: `y_pdf = H - y_spec`.
    51→
    52→---
    53→
    54→## A0.1 - Absolute Anchor Grid (Replaces Flow Layout)
    55→
    56→**Old (DEPRECATED):** `Y = previous_element_Y + height + spacing` - if one element overflows, everything below shifts and gets crushed.
    57→
    58→**New (MANDATORY):** Page height = 100%. Every element group gets an **absolute percentage Y-anchor**. Elements may only grow **within their own bounding box**. They NEVER push or compress other blocks.
    59→
    60→### Implementation Rule
    61→
    62→```
    63→# WRONG - flow layout (banned)
    64→y_cursor = PAGE_H - top_margin
    65→y_cursor -= title_height
    66→y_cursor -= gap
    67→y_cursor -= subtitle_height   # ← if title wraps, subtitle gets crushed
    68→
    69→# RIGHT - absolute anchor grid
    70→ANCHOR_TITLE_Y  = H * 0.30
    71→ANCHOR_SUMMARY_Y = H * 0.50
    72→ANCHOR_META_Y   = H * 0.70
    73→ANCHOR_FOOTER_Y = H * 0.90
    74→# Each block renders at its anchor, independent of others
    75→```
    76→
    77→### Bounding Box Containment
    78→
    79→Each block has a **maximum bounding box** defined by two consecutive anchors:
    80→- Block can grow downward within `[own_anchor_Y, next_anchor_Y - min_gap]`
    81→- If content exceeds its bounding box → trigger overflow protection (see Part 3)
    82→- Blocks NEVER consume space belonging to adjacent blocks

    86→## A0.2 - Typography Weight & Spacing System
    87→
    88→Use **weight**, **letter-spacing**, and **opacity** to create hierarchy - not just font size.
    89→
    90→### Mandatory Type Roles
    91→
    92→| Role | Size | Weight | Letter-Spacing | Line-Height | Opacity | Purpose |
    93→|------|------|--------|----------------|-------------|---------|---------|
    94→| **Kicker / Footer** (decorative text) | 16pt | Regular | 3pt (very wide) | - | 60% | Wide spacing + transparency makes 16pt text feel delicate and recessive |
    95→| **Summary / Description** (summary paragraph) 🆕 | 16-18pt | Regular | normal | **1.6** | 85% | **Fill visual space** - 2-4 lines of descriptive text that prevents empty covers |
    96→| **Meta / Subtitle** (secondary text) | 20-22pt | Light / Regular | normal | 1.4 | 85% | Comfortable reading rhythm, clear secondary hierarchy |
    97→| **Hero Title** (main title) | 45-65pt (CJK: 50-80pt) | Black / Heavy (extra bold) | normal-tight | **1.15** (multi-line) | 100% | Must create overwhelming scale contrast; visually dominates the page. CJK characters need +15-20% size to match Latin visual weight |
    98→
    99→### 🔴 Data-to-Drawer Binding Rule
   100→
   101→> **Hero Title = Company/entity name. Kicker = Report type/subtitle. Never reverse.**
   102→
   103→When users provide structured information (company name + report name/type), they must be bound to typography drawers by these rules:
   104→
   105→| User Data Field | Bound to Typography Role | Notes |
   106→|-------------|-------------|------|
   107→| **Company/entity name** (e.g. "GREENTECH") | **Hero Title** (45-65pt Heavy) | Company name is the **absolute visual center**, largest font, heaviest weight |
   108→| **Report type/subtitle** (e.g. "2025 Annual Report Summary") | **Kicker** (16pt, letter-spacing 3pt, opacity 60%) | Report name is decorative supplementary text, placed in small text position at top-left/above title |
   109→| **Summary/description** | **Summary** (16-18pt) | Detailed description text |
   110→| **Date/author/version** | **Meta** (20-22pt) | Auxiliary information |
   111→| **Document number/org signature** | **Footer** (16pt, opacity 60%) | Bottom closing |
   112→
   113→**Mapping priority (when ambiguous):**
   114→1. If user provides only one name → treat as company/entity name, bind to Hero Title
   115→2. If user provides two names → shorter/more brand-like → Hero Title; longer/descriptive → Kicker or Summary
   116→3. If user explicitly labels "title" and "subtitle" → title → Hero Title, subtitle → Kicker
   117→4. **Never use report type names (e.g. "Annual Report", "White Paper") as Hero Title's largest text** - report type is always Kicker-level decorative text
   118→
   119→### The Summary Block Rule (Anti-Void Iron Rule) 🆕
   120→
   121→> Every cover MUST include a Summary/Description text block. If the user provides no summary, the system MUST auto-generate one.
   122→
   123→**Why:** A cover with only a title and date looks barren. The Summary block physically fills 2-4 lines of space, preventing the "empty field" aesthetic.
   124→
   125→**Auto-generation rule:** When no summary/description is provided:
   126→```python
   127→# Generate a default summary
   128→if not summary_text:
   129→    summary_text = f"This report was generated by {org_name or 'the system'}, containing comprehensive data analysis and insights."
   130→    # Or in English:
   131→    # summary_text = f"This report presents comprehensive analysis and key insights prepared by {org_name or 'the organization'}."
   132→```
   133→
   134→**Constraints:**
   135→- Width: template-specific (typically `W * 0.5` to `W * 0.6`)
   136→- Lines: 2-4 lines (auto-wrap at width boundary)
   137→- Never truncate summary - if too long, reduce to 4 lines max with `...`
   138→
   139→### Font Weight Fallback
   140→
   141→- If font family lacks Black/Heavy weight → use Bold + slightly larger size (+4pt)
   142→- If font family lacks Light weight → use Regular + increased letter-spacing (+1pt)
   143→- CJK fonts: STSong-Light for Regular/Light roles; for Heavy/Black → increase size by 15% to compensate for single-weight CJK fonts
   144→- English kickers/footers: **FORCE UPPERCASE** via code (`text.upper()` / `toUpperCase()`)

   148→## A0.3 - Z-Index Layer Management
   149→
   150→All cover elements must be rendered in strict layer order. No exceptions.
   151→
   152→| Layer | Z-Index | Contents | Rules |
   153→|-------|---------|----------|-------|
   154→| **Layer 0** (base) | 0 | Background fill (white / light gray) | Always rendered first; full page |
   155→| **Layer 1** (background) | 1 | Grids, watermark letters, decorative blocks, large clipped graphics | **MUST enable clip-path** - background elements may extend beyond logical bounds but must be clipped to page physical bounds. Never let background elements inflate PDF page size. |
   156→| **Layer 2** (structure) | 2 | Ultra-thin divider lines, sidebars, corner crop marks | Structural guides that define spatial zones |
   157→| **Layer 3** (content) | 3 | All readable text content | Rendered last, always on top |
   158→
   159→### Clip-Path Enforcement
   160→
   161→> **Since V2.1, all covers are rendered via HTML/CSS.** The canonical clip pattern is CSS `overflow: hidden`. The ReportLab Python example below is kept as legacy reference for body-page background elements only.
   162→
   163→```css
   164→/* HTML/CSS cover (CANONICAL): clip background overflow */
   165→.cover-bg-layer {
   166→  position: absolute;
   167→  inset: 0;
   168→  overflow: hidden;  /* MANDATORY */
   169→  z-index: 1;
   170→}
   171→```
   172→
   173→```python
   174→# ReportLab (legacy reference, body pages only): clip background elements to page bounds
   175→canvas.saveState()
   176→p = canvas.beginPath()
   177→p.rect(0, 0, W, H)
   178→canvas.clipPath(p, stroke=0)
   179→# ... render background elements (Layer 1 ONLY) ...
   180→canvas.restoreState()
   181→```
   182→
   183→> ⚠️ **Clip scope = Layer 1 ONLY。** In HTML/CSS covers, `overflow: hidden` must ONLY be on the Layer 1 background container. Layer 2 (lines) and Layer 3 (text) containers must NOT have `overflow: hidden`.
   184→> For ReportLab body pages: `saveState()`/`restoreState()` must close immediately after Layer 1 background rendering.
   185→> Layer 2 (lines) and Layer 3 (text) must never be rendered within a clip scope, otherwise text will be clipped.
   186→
   187→### 🔴 Anti-Clip Bug (Layer 3 Text Truncation Fix)
   188→
   189→**Symptom:** Cover text is visible but truncated by an invisible boundary, only half visible.
   190→
   191→**Root cause:** clip/overflow scope not closed in time, causing subsequently rendered text to be clipped by the same clip rect.
   192→
   193→**Iron rule (HTML/CSS - canonical cover implementation):**
   194→```html
   195→<!-- ✅ CORRECT - overflow:hidden only on Layer 1 -->
   196→<div class="cover-layer-1" style="position:absolute; inset:0; overflow:hidden; z-index:1;">
   197→  <!-- Background decorative elements -->
   198→</div>
   199→<div class="cover-layer-2" style="position:absolute; inset:0; z-index:2;">
   200→  <!-- Structure lines, no overflow:hidden -->
   201→</div>
   202→<div class="cover-layer-3" style="position:absolute; inset:0; z-index:3;">
   203→  <!-- Text content, no overflow:hidden -->
   204→</div>
   205→
   206→<!-- ❌ WRONG - global overflow:hidden clips text -->
   207→<div class="cover-container" style="overflow:hidden;">
   208→  <div class="layer-1">...</div>
   209→  <div class="layer-2">...</div>
   210→  <div class="layer-3">...</div>  <!-- Text gets clipped! -->
   211→</div>
   212→```
   213→
   214→**Iron rule (ReportLab - body page background element reference):**
   215→```python
   216→# ✅ CORRECT - clip only wraps Layer 1
   217→canvas.saveState()
   218→canvas.clipPath(page_clip, stroke=0)
   219→render_layer_1_background(canvas)   # Background decoration
   220→canvas.restoreState()                # ← Must close here immediately!
   221→
   222→render_layer_2_lines(canvas)         # Structure lines - not inside clip
   223→render_layer_3_text(canvas)          # Text content - not inside clip
   224→
   225→# ❌ WRONG - text clipped by clip scope
   226→canvas.saveState()
   227→canvas.clipPath(page_clip, stroke=0)
   228→render_layer_1_background(canvas)
   229→render_layer_2_lines(canvas)         # ← Gets clipped
   230→render_layer_3_text(canvas)          # ← Gets clipped! Text only half visible
   231→canvas.restoreState()
   232→```
   233→
   234→```css
   235→/* Creative (HTML): Same principle */
   236→.cover-bg-layer  { overflow: hidden; z-index: 1; }  /* clip only on background layer */
   237→.cover-line-layer { overflow: visible; z-index: 2; } /* lines not clipped */
   238→.cover-text-layer { overflow: visible; z-index: 3; } /* text not clipped */
   239→```
   240→
   241→### 🔴 No Page Border/Frame
   242→
   243→**Symptom:** A rectangular border appears around the entire cover page, looking like a table.
   244→
   245→**Root cause:** ReportLab's `Frame()` defaults to `showBoundary=0`, but if set to `1` or `True`, it shows a border. Also `canvas.rect()` may accidentally draw a full-page rectangle.
   246→
   247→**Iron rule:**
   248→```python
   249→# Cover page Frame must have showBoundary=0
   250→Frame(x, y, w, h, showBoundary=0)  # Always 0
   251→
   252→# Never draw a full-page border on the cover
   253→canvas.rect(0, 0, W, H)  # ❌ BANNED on cover page
   254→
   255→# If using doc.showBoundary, cover page must be skipped
   256→doc = SimpleDocTemplate(..., showBoundary=0)  # Always 0 in production
   257→```
   258→
   259→```css
   260→/* Creative: No outer border on covers */
   261→.cover-page {
   262→  border: none !important;
   263→  outline: none !important;
   264→  box-shadow: none !important;
   265→}
   266→```
   267→
   268→### 🔴 Minimum Spacing Between Decorative Lines and Text (Line-to-Text Spacing)
   269→
   270→**Symptom:** Decorative lines on the cover (Layer 2 dividers, corner marks, sidebar edges) are flush against or overlapping with text.
   271→
   272→**Iron rule:**
   273→```
   274→Minimum spacing between decorative lines and any text content = U (= W * 0.05)
   275→i.e., at least 1 U of whitespace between line edges and text edges
   276→```
   277→
   278→| Line Type | Minimum Spacing | Notes |
   279→|---------|---------|------|
   280→| Horizontal divider | `U` above and below the line | Line must not be flush against title or body text |
   281→| Vertical sidebar edge | `U` to the right of the line | Text inside sidebar must maintain spacing from the edge |
   282→| Corner marks / crop marks | `1.5 * U` from mark endpoint to nearest text | Marks must not touch text |
   283→| Ultra-thick anchor line (Template 01) | `2 * U` to the right of the line | Thick line and title need ample breathing room |
   284→
   285→```python
   286→# Example: Template 01 vertical thick line to title spacing
   287→thick_line_x = 0.10 * W
   288→title_x = thick_line_x + 2 * U  # 2U spacing to the right of the thick line
   289→# ❌ WRONG: title_x = thick_line_x + 5  # 5pt is too close, will overlap
   290→```

   304→## Template 01: HUD Data Terminal - Ultra-Thick Vertical Anchor Line
   305→
   306→**Design intent:** A single bold vertical line on the left anchors all visual weight. The thick line eliminates left-side floating. Clean, data-driven, authoritative.
   307→
   308→### Layer 1 - Background
   309→- Full-page grid pattern: horizontal + vertical lines at ~50pt intervals
   310→- Grid color: primary color at **2% opacity** (white background, nearly invisible)
   311→- Grid line width: `0.5pt`
   312→
   313→### Layer 2 - Structure
   314→- **Left anchor line:** Start `(0.12*W, 0.1*H)`, End `(0.12*W, 0.9*H)`. Line width = **6pt**, primary color.
   315→- **Meta separator line:** At `Y_meta - 10pt`, from `X_content` to `X_content + W*0.4`, line width = **1pt**, primary color at 40% opacity.
   316→
   317→### Layer 3 - Content
   318→
   319→**Content left edge: `X_content = 0.12*W + 30pt`** (offset from the thick line)
   320→
   321→| Drawer | Y-Anchor | Content | Constraints |
   322→|--------|----------|---------|-------------|
   323→| **A - Kicker** | `0.15 * H` | Report type / subtitle (e.g. "2025 Annual Report Summary") | 16pt, Regular, letter-spacing 3pt, opacity 60%, uppercase |
   324→| **B - Hero Title** | `0.30 * H` | **Company/entity name** (e.g. "GREENTECH") | 45-65pt (CJK: 50-80pt), Heavy. Company name is the visual center |
   325→| **C - Summary** 🆕 | `0.50 * H` | 2-3 lines descriptive text about the report | 16-18pt, Regular, line-height 1.6, opacity 85%. **Width limit: `W * 0.6`**, auto-wrap. This drawer fills the mid-page void |
   326→| **D - Meta/Date** | `0.75 * H` | Author, org, date | 16-20pt, Regular. Top edge separated by the 1pt meta line |
   327→
   328→### Best For
   329→Technology reports, data analysis, dashboard summaries, technical white papers

   333→## Template 02: Corporate Editorial - Top Bar with Bottom Accent
   334→
   335→**Design intent:** Top-bottom symmetry. Top bar provides structural weight, bottom-right info block creates diagonal balance. Solves the "empty edges" problem.
   336→
   337→### Layer 1 - Background
   338→- **Background giant year watermark:** Text = current year (e.g. "2026"), **Max font size = 180pt**, measure rendered width - if it exceeds `W * 0.85`, scale down proportionally. Position: `X = W - 20pt` (right edge), `Y = 0.15*H`. Color = primary at **4% opacity**. Font weight = Black. ⚠️ **Full-display iron rule: watermark text must be 100% within the visible page area - cropping is strictly forbidden. Prefer reducing font size over truncation.**

   345→
   346→| Drawer | Position | Content | Constraints |
   347→|--------|----------|---------|-------------|
   348→| **Left upper - Title group** | `X = 0.12*W`, `Y = 0.15*H` | Kicker (report type/subtitle, 16pt) → Hero Title (company/entity name, 45-65pt / CJK 50-80pt, Heavy) | Stack downward from anchor |
   349→| **Mid-left - Summary** 🆕 | `X = 0.12*W`, `Y = 0.50*H` | Descriptive paragraph | 16-18pt, Regular, line-height 1.6. **Width limit: `W * 0.5`** |
   350→| **Right lower - Meta** | Right-aligned at `X = 0.88*W - 20pt`, `Y = 0.70*H` | Date, version, author | **Right-aligned**, 16-20pt. Must hug the 4pt accent line |

   357→## Template 03: The Monolith - Hard-Left Alignment + Right-Side Giant Watermark Counterweight
   358→
   359→**Design intent:** Everything hard-left. Right-side watermark counterbalances the asymmetry. Solves the "right half is empty" bug.

   361→### Layer 1 - Background

Anchor at `X = 0.85*W`, vertically centered: `Y = (H - rendered_text_width) / 2`. Color = primary at **3% opacity**. ⚠️ **Full-display iron rule: watermark text must be 100% within the visible page area - cropping is strictly forbidden. Prefer reducing font size over truncation.**
   363→
   364→### Layer 2 - Structure
   365→- **Color dash (visual guide line):** At `(0.12*W, 0.15*H)`, draw a horizontal bar: width = **50pt**, height = **5pt**, primary color.

   368→### Layer 3 - Content
   369→
   370→**Unified left edge: `X = 0.12*W`**
   371→
   372→| Drawer | Y-Anchor | Content | Constraints |

   377→| **D - Summary** 🆕 | `0.45 * H` | Descriptive paragraph (key anti-void element) | 16-18pt, Regular, line-height 1.6. **Width limit: `W * 0.55`** (must not collide with right watermark) |

   386→## Template 04: Museum Minimal - Refined Corner Crop Marks
   387→
   388→**Design intent:** Abandon all-over scattered layout. Four corner crop marks form an invisible "force field box" that concentrates all content dead center.
   389→
   390→### Layer 2 - Structure
   391→- Set safety margin `M = 0.08 * W`
   392→- **Four corner marks** at inner corners: `(M, M)`, `(W-M, M)`, `(M, H-M)`, `(W-M, H-M)`
   393→- Each mark: L-shaped, arm length = **30pt**, line width = **2pt**, primary color at 60% opacity
   394→- Marks point **inward** (top-left: right arm + down arm)
   395→
   396→### Layer 3 - Content
   397→
   398→**This template FORBIDS hardcoded absolute Y coordinates.**
   399→
   400→**Centering algorithm (mandatory):**
   401→1. Pre-compose ALL text elements (kicker + title + summary + meta) into a single virtual Text Block
   402→2. Calculate the block's total rendered height `Block_H`
   403→3. Position block: `X = 0` (full width, center-aligned text), `Y = (H - Block_H) / 2`
   404→4. **This guarantees the content group is vertically centered regardless of how much content there is**
   405→
   406→**Internal spacing within the centered block:**
   407→- Kicker → Title: `24pt`
   408→- Title → Summary: `20pt`
   409→- Summary → Meta: `40pt`
   410→
   411→### Type Scale
   412→| Role | Size | Notes |
   413→|------|------|-------|
   414→| Kicker | 16pt | Uppercase, letter-spacing 4pt, opacity 50%. Bound to report type/subtitle |
   415→| Hero Title | 48-60pt | Heavy - slightly smaller than other templates to fit center composition. **Bound to company/entity name** |
   416→| Summary 🆕 | 16-18pt | Regular, line-height 1.6, center-aligned, width ≤ `W * 0.6` |
   417→| Meta | 16pt | Regular, opacity 60%, at bottom of group |
   418→
   419→### Best For
   420→Gallery catalogs, design portfolios, exhibition materials, luxury brand documents

   424→## Template 05: Floating Diagonal - Premium Whitespace with Binding Line
   425→
   426→**Design intent:** "Left-upper to right-lower" diagonal visual flow. The two text groups create tension across whitespace. The gap IS the design.
   427→
   428→### Layer 2 - Structure
   429→- **Binding dashed line:** At `X = 0.08*W`, from `Y = 0.05*H` to `Y = 0.95*H`. Line width = **1pt**, dashed (dash 6pt, gap 8pt), color = light gray (#d0d0d0, 40% opacity).
   430→
   431→### Layer 3 - Content
   432→
   433→| Group | Position | Content | Constraints |
   434→|-------|----------|---------|-------------|
   435→| **Upper-left group** | Anchor: `X = 0.15*W`, `Y = 0.20*H` | Kicker (report type/subtitle, 16pt gap) → Hero Title (company/entity name, 45-65pt / CJK 50-80pt, Heavy) | Left-aligned. Width limit: `W * 0.7` |

   445→## Template 06: Swiss Grid - Ultimate Precision, Curing All Misalignment
   446→
   447→**Design intent:** The typographic "multiplication table." Thick lines physically slice the page into cells. Content fills its assigned cell. Impossible to misalign.
   448→
   449→### Layer 2 - Structure (ABSOLUTE - non-negotiable)
   450→
   451→```
   452→Horizontal line 1: (0.1*W, 0.25*H) → (0.9*W, 0.25*H), width 2pt, primary
   453→Horizontal line 2: (0.1*W, 0.75*H) → (0.9*W, 0.75*H), width 2pt, primary
   454→Vertical line 1:   (0.45*W, 0.25*H) → (0.45*W, 0.75*H), width 2pt, primary
   455→```
   456→
   457→These create 4 zones:
   458→
   459→```
   460→┌──────────────────────────────────────┐
   461→│         Zone A - Top Strip           │  ← Kicker / report type
   462→├──────────────────┬───────────────────┤
   463→│   Zone B         │   Zone C          │
   464→│   (left cell)    │   (right cell)    │  ← B: Hero Title (MUST fill)
   465→│   X: 0.1W-0.43W │   X: 0.48W-0.9W  │  ← C: Summary text (MUST fill)
   466→├──────────────────┴───────────────────┤
   467→│         Zone D - Bottom Strip        │  ← Footer / year / doc number
   468→└──────────────────────────────────────┘
   469→```
   470→
   471→### Layer 3 - Content (STRICT zone containment)
   472→
   473→| Zone | Content | X Range | Y Range | Notes |
   474→|------|---------|---------|---------|-------|
   475→| **A** | Kicker / report type | `0.10*W` - `0.90*W` | `0.15*H` - `0.23*H` | Left-aligned at `X = 0.12*W` |
   476→| **B** | **Hero Title (company/entity name)** | `0.10*W` - `0.43*W` | `0.28*H` - `0.70*H` | **Width = `0.33*W`**. Font must be large enough to physically fill the cell. Text wraps at boundary. |
   477→| **C** | **Summary text** 🆕 | `0.48*W` - `0.90*W` | `0.28*H` - `0.70*H` | **Must contain substantial descriptive text** - this zone MUST be filled. 16-18pt, Regular, line-height 1.6. This is the primary anti-empty-page mechanism. |
   478→| **D** | Footer / date / number | `0.10*W` - `0.90*W` | `0.78*H` - `0.88*H` | Can split: left part + right-aligned part |
   479→
   480→### Zone Overflow Protection (MANDATORY)
   481→
   482→If text in Zone B or C exceeds the vertical boundary (Y > `0.70*H`):
   483→1. **Step 1:** Reduce font size by 2pt increments (minimum: 16pt for summary, 40pt for title)
   484→2. **Step 2:** If still overflows, truncate with `...` ellipsis
   485→3. **NEVER** let text cross a grid line - the grid is sacred
   486→
   487→**Hard width enforcement:**
   488→```python
   489→# Zone B: title MUST wrap within its cell width
   490→zone_b_max_width = 0.33 * W
   491→# If title renders wider → word-wrap, NEVER let it bleed into Zone C
   492→
   493→# Zone C: summary MUST wrap within its cell width
   494→zone_c_max_width = 0.42 * W  # (0.90 - 0.48) * W
   495→# Wrap at boundary, add lines, NEVER cross the vertical grid line
   496→```
   497→

   499→Swiss-style design, data-heavy reports, structured corporate documents, annual reports

   588→## S3.1 - Hero Title Overflow Protection (Title Line-Wrapping)
   589→
   590→**Rule:** Hero title must NEVER exceed its template's width boundary.
   591→
   592→**Algorithm:**
   593→1. Measure the rendered width of the hero title string at the target font size
   594→2. If `rendered_width > max_width`:
   595→   - Word-wrap at boundary (CJK: any character; Latin: space/hyphen; Mixed: CJK/Latin boundaries)
   596→   - Multi-line hero titles: **lock line-height to 1.15**
   597→3. Maximum lines: **3** - if title needs 4+ lines, reduce font size by 4pt increments until ≤ 3 lines
   598→4. Minimum font size floor: **40pt** (below this, truncate with `...`)
   599→
   600→```python
   601→def safe_hero_title(text, font, max_size, max_width, min_size=40):
   602→    size = max_size
   603→    while size >= min_size:
   604→        lines = word_wrap(text, font, size, max_width)
   605→        if len(lines) <= 3:
   606→            return lines, size
   607→        size -= 4
   608→    return truncate_with_ellipsis(text, font, min_size, max_width, max_lines=3), min_size
   609→```
   610→

   613→## S3.2 - Zone Collision Detection
   614→
   615→After rendering each text block, check if its bottom edge penetrates the next zone boundary:
   616→
   617→1. **Step 1 - Font reduction:** Decrease by 2pt. Floor: 16pt for meta/summary, 40pt for titles.
   618→2. **Step 2 - Truncation:** If font reduction fails, truncate with `...`
   619→3. **Step 3 - Log warning:** Output a warning about content truncation
   620→
   621→```python
   622→def enforce_zone_bounds(text, font, size, zone_y_max, min_size=16):
   623→    while size >= min_size:
   624→        rendered_height = measure_text_height(text, font, size)
   625→        if current_y + rendered_height <= zone_y_max:
   626→            return text, size
   627→        size -= 2
   628→    return truncate_to_fit(text, font, min_size, zone_y_max - current_y), min_size
   629→```

   653→
   654→**Every drawer/zone has a maximum width. Text wrapping MUST respect this width exactly.**

   687→## S3.6 - Background Watermark Full-Display Enforcement 🆕
   688→
   689→**All watermark text in the background layer (Layer 1) must be 100% within the visible page area. Cropping, truncation, or extending beyond page boundaries is strictly forbidden.**
   690→
   691→**Applicable scope:**
   692→- Template 02 giant year watermark
   693→- Template 03 right-side vertical watermark
   694→- Template 07 sidebar watermark
   695→- `cover-backgrounds.md` Recipe 2.1 giant sidebar pillar
   696→- `cover-backgrounds.md` Recipe 2.2 bottom full-size text
   697→- Any other decorative text in the background layer
   698→
   699→**Adaptive algorithm (mandatory):**
   700→
   701→```python
   702→def safe_watermark_size(text, font, max_size, available_space):
   703→    """
   704→    Ensure watermark text is fully displayed within available space.
   705→    available_space: available width/height (depending on text direction)
   706→    """
   707→    rendered = measure_text(text, font, max_size)
   708→    if rendered > available_space:
   709→        return max_size * (available_space / rendered)
   710→    return max_size
   711→```
   712→
   713→**Rules:**
   714→1. Horizontal text: rendered width must not exceed `W * 0.90` (5% safety margin on each side)
   715→2. Vertical/rotated text: rendered height must not exceed `H * 0.85` (7.5% safety margin top and bottom)
   716→3. If exceeded, scale down font size proportionally - never truncate
   717→4. **Anchor coordinates must never exceed page boundaries** (no negative X/Y values or values exceeding W/H)
   718→
   719→**This is a visual quality red line: a truncated "REPO" is worse than no watermark at all. A complete "REPORT" is the design.**

   723→## S3.7 - Line-Length Alignment (Line Must Match Text Span)
   724→
   725→**Problem:** Decorative lines (vertical accent lines, horizontal dividers, underlines) are arbitrary lengths that don't relate to the text they accompany, creating visual disconnect.
   726→
   727→**Iron Rule:** Lines must be sized relative to the text they serve:
   728→
   729→### Vertical Lines (e.g., Template 01 thick line, Template 08 accent line)
   730→
   731→**Vertical line height = text block height** (from first text element to last text element in the same column).
   732→
   733→```
   734→# WRONG - arbitrary fixed height
   735→vline_top = 0.1 * H
   736→vline_bottom = 0.9 * H  # ← line runs full page regardless of content
   737→
   738→# RIGHT - measure text block, then draw line
   739→text_top = first_element_y        # e.g., label at 0.12*H
   740→text_bottom = last_element_y + last_element_height  # e.g., footer at 0.88*H
   741→vline_top = text_top - U           # 1U padding above first element
   742→vline_bottom = text_bottom + U     # 1U padding below last element
   743→```
   744→
   745→### Horizontal Lines (e.g., Template 08 hline, dividers)
   746→
   747→**Horizontal line width ≥ text width of the widest text element in its zone.** Lines may be slightly longer (up to 120% of text width) but NEVER shorter.
   748→
   749→```python
   750→# WRONG - fixed short line
   751→hline_width = 200  # ← might be shorter than the title
   752→
   753→# RIGHT - measure, then draw
   754→max_text_width = max(measure(title), measure(subtitle), measure(authors))
   755→hline_width = max(max_text_width, max_text_width * 1.1)  # at least as wide, up to 110%
   756→# Clamp to available space
   757→hline_width = min(hline_width, available_width)
   758→```
   759→
   760→### HTML/CSS Implementation
   761→
   762→For HTML/Playwright covers, use relative sizing:
   763→```css
   764→/* Vertical line spans the content block */
   765→.vline {
   766→  position: absolute;
   767→  top: var(--content-top);     /* align with first text element */
   768→  bottom: var(--content-bottom); /* align with last text element */
   769→}
   770→
   771→/* Horizontal divider: min-width matches text container */
   772→.hline {
   773→  width: max(100%, 200px);     /* at least as wide as parent text container */
   774→}
   775→```
   776→
   777→**Checklist:**
   778→- [ ] Every vertical line's height matches its adjacent text block span (± 1U padding)
   779→- [ ] Every horizontal line's width ≥ widest text element in its zone
   780→- [ ] No decorative line is shorter than the text it accompanies

   784→## S3.8 - Vertical Balance (Anti-Top-Heavy Layout)
   785→
   786→**Problem:** Content clusters at the top of the page, leaving the bottom 40%+ as dead whitespace. This happens when anchor points are set too high and don't adapt to content volume.
   787→
   788→**Root cause:** Fixed anchor grid with `ANCHOR_TITLE_Y = 0.20*H` pushes everything upward regardless of how much content there is.
   789→
   790→### Solution: Adaptive Vertical Centering
   791→
   792→**When total content height < 50% of page height, switch to centered distribution mode:**
   793→
   794→```python
   795→# Step 1: Calculate total content height
   796→content_elements = [title, subtitle, summary, meta, footer]
   797→total_content_h = sum(elem.height for elem in content_elements) + total_gaps
   798→
   799→# Step 2: Check fill ratio
   800→fill_ratio = total_content_h / (H * 0.80)  # usable height (excluding margins)
   801→
   802→if fill_ratio < 0.50:
   803→    # LOW CONTENT MODE - vertically center the entire block
   804→    start_y = (H - total_content_h) / 2
   805→    # Distribute elements from start_y downward with standard gaps
   806→else:
   807→    # NORMAL MODE - use anchor grid
   808→    # But shift anchors down: title at H*0.30-0.35 (not 0.20-0.25)
   809→    pass
   810→```
   811→
   812→### Anchor Adjustment Rules
   813→
   814→| Content Volume | Title Anchor | Summary Anchor | Meta Anchor |
   815→|---------------|-------------|----------------|-------------|
   816→| **Sparse** (fill < 50%) | Centered mode | Centered mode | Centered mode |
   817→| **Normal** (fill 50-80%) | `H * 0.30` | `H * 0.48` | `H * 0.70` |
   818→| **Dense** (fill > 80%) | `H * 0.20` | `H * 0.40` | `H * 0.65` |
   819→
   820→### CJK Title Size Compensation
   821→
   822→CJK characters at the same pt size as Latin characters appear visually smaller due to denser stroke structure. Compensate:
   823→
   824→```
   825→CJK Hero Title:    50-80pt (Latin: 45-65pt) - increase by 15-20%
   826→CJK Kicker:        11-12pt (Latin: 9pt)
   827→CJK Summary:       17-20pt (Latin: 16-18pt)
   828→```
   829→
   830→**Detection:** If title string contains CJK characters (`\u4e00-\u9fff`), apply CJK size multiplier.
   831→
   832→### HTML/CSS Implementation
   833→
   834→```css
   835→/* Vertical centering mode for sparse content */
   836→.cover.sparse-content .center-block {
   837→  justify-content: center;  /* flexbox vertical center */
   838→}
   839→
   840→/* CJK title size bump */
   841→.title:lang(zh), .title:lang(ja), .title:lang(ko) {
   842→  font-size: clamp(50pt, 8vw, 80pt);  /* larger than Latin range */
   843→}
   844→```
   845→
   846→**Checklist:**
   847→- [ ] No cover has >40% dead whitespace at the bottom
   848→- [ ] Content is visually centered on the page (optical center, not mathematical)
   849→- [ ] CJK titles are 15-20% larger than equivalent Latin titles
   850→- [ ] Sparse-content covers use centered distribution, not fixed anchors

   854→## S3.9 - Percentage Positioning Requires Known-Size Container
   855→
   856→**Root cause of bad case:** A wrapper div (e.g. `.content-left`) with `position: absolute` but **no explicit height** contains children positioned with `top: XX%`. CSS percentage `top` resolves against the containing block's **height** — if that height is zero or undefined (because all children are also absolutely positioned, contributing no content height), the percentage values collapse and elements stack on top of each other.
   857→
   858→**Iron rule:** When using `top: XX%` (or `bottom: XX%`) to position child elements, the containing block MUST have a **deterministic height** — one of:
   859→
   860→| Method | Example | When to use |
   861→|--------|---------|-------------|
   862→| Explicit `height` | `height: 100%` or `height: var(--h)` | Wrapper spans full page |
   863→| `top` + `bottom` pair | `top: 0; bottom: 0;` | Wrapper stretches between two edges |
   864→| `inset: 0` | `inset: 0;` | Shorthand for full-page wrapper |
   865→
   866→**Preferred pattern — flat structure with px values (safest):**
   867→```css
   868→/* ✅ CORRECT: children positioned directly in .cover with px values */
   869→.cover { position: relative; width: 794px; height: 1123px; }
   870→.kicker   { position: absolute; top: 225px;  left: 95px; }
   871→.title    { position: absolute; top: 292px;  left: 95px; }
   872→.summary  { position: absolute; top: 539px;  left: 95px; }
   873→.meta     { position: absolute; top: 786px;  left: 95px; }
   874→```
   875→
   876→**Acceptable — wrapper with deterministic height:**
   877→```css
   878→/* ✅ OK: wrapper has inset:0, so height = parent height = 1123px */
   879→.content-left { position: absolute; inset: 0; width: 55%; }
   880→.title   { position: absolute; top: 26%; }  /* 26% of 1123px = 292px ✓ */
   881→```
   882→
   883→**Forbidden — wrapper with no height:**
   884→```css
   885→/* ❌ BANNED: .content-left has no height/bottom, percentage top is undefined */
   886→.content-left { position: absolute; left: 12%; top: 0; width: 55%; }
   887→.title   { position: absolute; top: 26%; }  /* 26% of WHAT? → collapse → overlap */
   888→.summary { position: absolute; top: 48%; }  /* stacks on top of title */
   889→```
   890→
   891→**Quick self-check before writing cover CSS:**
   892→1. For every element with `top: XX%` — trace upward: does the containing block have a known height?
   893→2. If unsure → use `px` values instead (calculate from `var(--h)` manually: `26% × 1123 = 292px`)
   894→3. If using a grouping wrapper → give it `inset: 0` or explicit `height: 100%`

   905→Cover background = Pure white / very light gray / primary at 5-8% opacity

   913→- ❌ Rainbow / multi-color gradients
   914→- ❌ Dense textures or patterns
   915→- ❌ Piling on decorative elements - restraint > clutter
   916→- ❌ More than 2 typefaces on a cover
   917→- ❌ Centered text + gradient/solid background (PowerPoint aesthetic)

  1384→**Layout rules:**
  1385→1. **Border frame**: 2.5pt solid black, inset ~5% from all page edges (40px left/right, 56px top/bottom on A4 at 96dpi). This is the defining visual element.
  1386→2. **Institution name**: Centered, serif, 28-34pt Bold, letter-spacing 4-6px. For CJK names, use wider letter-spacing (6px). For Latin names, use standard (3px).
  1387→3. **Thick divider**: 2pt solid line, 70% width, separates institution from content below.
  1388→4. **Document type**: 20-24pt, lighter weight than institution name, letter-spacing 3-4px. This slot differentiates document categories (e.g. "开题报告" / "Thesis Proposal" / "毕业论文" / "Graduation Design").
  1389→5. **Title**: Serif, 26-30pt Bold, max 3 lines, centered. Auto-wrap at 520px width.
  1390→6. **Structured fields**: Left-aligned label + centered underline value. Label width fixed by longest label in the set. 3-7 field rows supported. Common fields: Name/姓名, Student ID/学号, Advisor/导师, Department/院系, Major/专业.
  1391→7. **Date**: Centered at bottom, 14pt, letter-spacing 2px.
  1392→
  1393→**Field auto-detection:**
  1394→When the user provides structured metadata (name, student ID, advisor, etc.), auto-populate the fields block. When no fields are provided, omit the fields-block entirely and let the title expand vertically into the freed space.
  1395→
  1396→**Variant 11B - Double border:**
  1397→For extra formality (government documents, official submissions), replace the single border with a double border (outer 2.5pt + inner 1pt, 6px gap):
  1398→```css
  1399→.border-frame {
  1400→  border: 2.5px solid var(--c-accent);
  1401→  outline: 1px solid var(--c-accent);
  1402→  outline-offset: 6px;
  1403→}
  1404→```
  1405→
  1406→**This template has NO background decoration layer, NO watermarks, NO gradients.** The black frame + white space IS the design.

  1410→### Academic Template Selection Guide
  1411→
  1412→| Scenario | Template | Rationale |
  1413→|----------|----------|-----------|
  1414→| arXiv preprint, technical report | **08** (Vertical Anchor) | Left-aligned, data-dense feel |
  1415→| IEEE/ACM paper, English thesis | **09** (Symmetric) | Classic bilateral symmetry |
  1416→| Chinese thesis, journal with keywords | **10** (Journal) | CJK-optimized, keyword block |
  1417→| **Thesis proposal, institutional cover, government doc** | **11** (Institutional) | **White bg, black border frame, structured field slots** |
  1418→| Light/formal academic (white bg) | **01-07** (standard templates) | Use standard cover system |
  1419→
  1420→
  1421→Covers support a **background decoration layer** rendered behind all foreground content (Layer 1). This layer adds subtle depth through supergraphics, typographic watermarks, and blueprint hairlines.
  1422→
  1423→See → `typesetting/cover-backgrounds.md` - complete specification with modules, recipes, and constraint matrix.
  1424→
  1425→**Quick reference:**
  1426→- **Recipe A (Minimalist Modern)**: Deep-space arc only - safest, max whitespace
  1427→- **Recipe B (Engineering Academic)**: Coordinate cross + vertical spine text - precision, engineering feel
  1428→- **Recipe C (Stable & Authoritative)**: Angle slash + bottom bleed text - heavy, authoritative
  1429→
  1430→**Background layer is OPTIONAL.** Not every cover needs one. Templates 01-07 already define their own Layer 1 backgrounds - use the recipes only when a template's built-in background is insufficient.
